悟已往之不谏，知来者之可追

一 今夕是何年

2017年伊始，有感于岁月不可蹉跎，志向需铭记于心，遂成文以记。

二 先进的技术可以极大的提高生产力

转眼北京工作已两年半了，2016在技术上的最大突破便是由业务转向架构，伴随着这个转变，获得了巨大的收获，同时也思维方式上也有所转变。

也在入职OFashion的前一年多时间里，专注于公司的业务开发，之后对工程里的业务已熟稔于心

在做业务所获得的最大的收获：

  1. 对现有条件进行合理的利用，发扬优势，规避缺陷
  2. 设计一套灵活，可配置的业务逻辑来面对复杂多变的产品需求
  3. 代码重构 [收获与教训同步]，突然想看看《Rework》那本书
  4. 业务的经验积累到一定程度的时候，相信每个工程师都会对现有的工程有自己的一份见解，随着公司的发展，在瑞哥的信任和鼓励之下，我开始了对公司现有的架构做重构工作

架构工作所带来的几点工作方式和思维方式的变化

  1. 思考“两小时”，编码“五分钟”

    框架的一点改变，对业务的影响有可能是灾难性，这是重构需要注意的点，同时也是设计需要的点。灵活的设计也是为面对未知的未来变化重要的特性，低耦合是非常好的实践。

  2. 分解

    一个新的模块的而建立，将其拆分成多个子功能，各司其职，降低模块的复杂度，这是软件设计的一个方法。在软件领域是一个很好的方法，但何尝不是一个【普世的方法论】？将复杂的任务或目标，拆分成一个一个小的任务或目标，去解决这一个一个问题，目标或任务就会在这个过程中逐步清晰，不再是遥不可及。由此做了整个项目标志性的变化，【组件化】。组件化的拆分目标是整个工程。

  3. 所为何

    框架的设计和演变是为人和需求而改变的，不是为框架本身而改变的。以人为出发点的技术，才能是真正的技术。

三 看山不是山

佛家有三重境，一为：看山是山，看水是水。二为：看山不是山，看水不是水。三为：看山是山，看水是水。若将对技术的追求对应着三种境界，我在走入“看山不是山，看水不是水”。也像王国维对人生所定义的第二重境界“衣带渐宽终不悔，为伊消得人憔悴”
