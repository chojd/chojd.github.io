CocoaPods安装（安装过程如果遇到问题，请查看常见错误）

使用gem(ruby包管理工具)，安装CocoaPods

查看gem源

gem sources -l

grass in ~ λ gem sources -l
*** CURRENT SOURCES ***

https://rubygems.org/

ruby默认源(https://rubygems.org/)被墙，先修改下默认源

移除默认源

gem sources –remove https://rubygems.org/

使用新的ruby源 ruby源淘宝镜像

gem sources -a https://ruby.taobao.org/

查看修改后的的源地址（请确保只有一个源地址）

gem source -l

grass in ~ λ gem source -l
*** CURRENT SOURCES ***

https://ruby.taobao.org/

安装cocoapods(苹果系统升级OS X EL Captian 安装命令需要修改,使用第二条命令)

sudo gem install cocoapods

sudo gem install -n /usr/local/bin cocoapods

构建CocoaPods

https://github.com/grassexplorer/PodLib

使用 Pod Lib Create 命令

pod lib create PodLib

命令说明：
    Usage:

        $ pod lib create NAME

          Creates a scaffold for the development of a new Pod named `NAME` according to
          the CocoaPods best practices. If a `TEMPLATE_URL`, pointing to a git repo
          containing a compatible template, is specified, it will be used in place of the
          default one.

    Options:

        --template-url=URL   The URL of the git repo containing a compatible template
        --silent             Show nothing
        --verbose            Show more debugging information
        --no-ansi            Show output without ANSI codes
        --help               Show help banner of specified command

创建之前回答几个问题，按提问选择就行（创建过程需要从远端下载模板，配置好后，工程会自动启动）

 > yes

Which testing frameworks will you use? [ Specta / Kiwi / None ]
 > none

Would you like to do view based testing? [ Yes / No ]
 > no

What is your class prefix?
 > OF

#常见错误#

错误1：

Error fetching http://ruby.taobao.org/:

bad response Not Found 404 (http://ruby.taobao.org/specs.4.8.gz)

解决方案：将gem源替换为https://ruby.taobao.org/

错误2：

ERROR: While executing gem … (Errno::EPERM)

Operation not permitted - /usr/bin/pod

解决方案：苹果系统升级OS X EL Capitan后会出现的插件错误，将安装流程 4.安装CocoaPods 的 (1)sudo gem install cocoapods ——>改为sudo gem install -n /usr/local/bin cocoapods
